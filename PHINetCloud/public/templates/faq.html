Frequently Asked Questions


  • How can I contact you?
    • Please send all queries to rseales@troy.edu.

  • Where can I find documentation?
    • Please refer to our github and its wiki: github.com/seales/PHINet.

  • What types of sensors can you support?
    • We currently only support a HeartBeat Sensor through the camera embedded into the device. Look for more sensors in the future.

  • What happens if content goes stale?
    • Given NDN, dynamic content may have the same name despite there being various versions. Many open questions still exist here. Currently we do nothing to ensure that content doesn't go stale. More sophisticated measures may exist in the future.

  • How do you determine the name of data to request?
    • Content naming, within PHINet, is deterministic due to each component of the name being known to exist. As such, we can easily construct all names without implementing longest prefix matching or some other method.

  • What are the implications of connecting via my mobile network versus connecting via WiFi?
    • This is a trouble area since NDN hasn't been deployed. Connecting via your mobile network will only allow communication with the server while connecting via WiFi makes it possible to communicate with both the server and any PHINet-enabled device over LAN.

  • Why can't doctors add patients?
    • The current solution (only patients can add doctors) was implemented first due to it being quick and simple - patients choose to reveal things, doctors cannot ask. However, more sophisticated mechanisms may be implemented in the future.
